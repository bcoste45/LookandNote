2- Besoins fonctionnels

    Cette partie décrit de manière structurée les fonctionalités du système. Pour chaque fonctionalité, une description narrative est donnée.

  • Connexion/déconnexion :

    • Inscription client :

      • Description :Un utilisateur s'inscrit en donnant un identifiant, une adresse mail valide et un mot de passe
      • Priorité : Obligatoire
      • Justification : L'utilisateur doit absolument s'inscrire pour avoir un compte Look&Note
      • Contraintes : Accès unique à partir de la page de connexion Look&Note

    • Connexion client :

      • Description :Un utilisateur se connecte en donnant son identifiant et son mot de passe. Après vérification, l'utilisateur passe en mode connecté
      • Priorité : Obligatoire
      • Justification : L'utilisateur doit absolument se connecter pour accéder à son espace personnel
      • Contraintes : Accès unique à partir de la page de connexion Look&Note

    • Déconnexion client:

      • Description :Un utilisateur se déconnecte en cliquant sur Déconnexion. Ceci le renvoie sur la page de connexion Look&Note
      • Priorité : Obligatoire
      • Justification : Si l'utilisateur souhaite se connecter sur un autre compte Look&Note, il devra se déconnecter d'abord de son compte courant
      • Contraintes : Accès à partir de toutes les pages du compte utilisateur

  • Publication :

    • Publier :

      • Description :L'utilisateur peut lui seul publier une photo ou une vidéo à partir de la page Publier de son profil et choisir des critères selon lesquels elle va être jugée, cette dernière peut être supprimée dans la page Profil du profil de l'utilisateur
      • Priorité : Obligatoire
      • Justification : L'utilisateur doit avoir la possibilité de publier des photos afin qu'elles soient jugées, ceci étant l'objectif même du site
      • Contraintes : Les amis de l'utilisateur n'ont pas le droit de publier une photo dans son profil

    • évaluations/commentaires :

      • Description :Les amis de l'utilisateur doivent avoir la possibilité de noter et de commenter ses photos via les pages Accueil et Profil de l'utilisateur. Ce dernier doit pouvoir à partir de la page évaluations commenter ses publications et consulter les commentaires ainsi que les notes qui lui ont été attribuées et celles qu'il a donné. L'évaluation se fait en attribuant une note : 1, 2, 3, 4 ou 5. L'utilisateur peut noter et commenter à son tour les photos de ses amis via les pages Accueil et Profil de ses amis
      • Priorité : Obligatoire
      • Justification : La possibilité de commenter et de noter les photos permet à l'utilisateur de connaître l'avis des ses amis et de leurs cotés de connaître le sien

  • Statistiques :

    • Statistiques par photo :

      • Description :L'utilisateur et ses amis doivent pouvoir consulter les statistiques d'une photo en cliquant dessus, celles-ci correspondent au nombre de personnes qui ont attribué une note de 1, 2, 3, 4 ou 5
      • Priorité : Obligatoire
      • Justification : Les statistiques par photo donnent un aperçu plus précis des notes attribuées
      • Contraintes : Ces statistiques sont visibles par l'utilisateur et ses amis

    • Statistiques par profil :

      • Description :L'utilisateur et ses amis doivent pouvoir consulter les statistiques d'une personne en se mettant sur la page Profil, ces dernières sont relatives à la moyenne des notes données, la moyenne des notes obtenues, le nombre de notes données, le nombre de notes obtenues et le nombre de publications
      • Priorité : Obligatoire
      • Justification : Les statistiques par profil donnent un aperçu sur le taux d'activité de l'utilisateur
      • Contraintes : Ces statistiques sont visibles par l'utilisateur et ses amis et l'accès se fait à partir de la page Publier

  • Amis :

    • Rechercher des amis :

      • Description :L'utilisateur a la possibilité de rechercher des amis
      • Priorité : Obligatoire
      • Justification : L'objectif est de rechercher des amis afin de pouvoir les ajouter parmi sa liste d'amis
      • Contraintes : Accès unique à partir de la page Amis

    • Ajouter des amis :

      • Description :L'utilisateur a la possibilité d'ajouter des amis à sa liste d'amis en leur envoyant une demande d'ajout
      • Priorité : Obligatoire
      • Justification : L'objectif est d'avoir plus d'amis
      • Contraintes : Accès unique à partir de la page Amis
      • Développement : Recherche par mots clés de la personne que l'utilisateur souhaite ajouter parmi ses amis puis envoi de la demande d'ajout. Celle-ci se met en attente et le destinataire reçois un message de l'expéditeur qui souhaite l'ajouter, si le destinataire accepte la demande de celui-ci, chacun ouvrira les droits à l'autre d'accéder de manière complète à son profil

    • Supprimer des amis :

      • Description : L'utilisateur a la possibilité de supprimer des amis de sa liste d'amis
      • Priorité : Obligatoire
      • Justification : L'objectif est de supprimer les personnes que l'utilisateur ne souhaite plus compter parmi sa liste d'amis
      • Contraintes : Accès unique à partir de la page Amis

  • Contact et législation :

    • A propos du site :

      • Description :Les utilisateurs peuvent accéder via la page A propos à une description détaillée du site Look&Note
      • Priorité : Obligatoire
      • Justification : L'objectif est de décrire à quoi sert le site afin d'attirer l'oeil des futurs utilisateurs
      • Contraintes : Accès unique à partir de la page de connexion Look&Note

    • Prendre contact :

      • Description :Un utilisateur peut prendre contact avec le site Look&Note via la page Nous contacter en fournissant des informations relatives à sa civilité, son adresse mail, son nom, son prénom, le sujet de contact et une description détaillée
      • Priorité : Obligatoire
      • Justification : L'objectif est de répondre aux interrogations des utilisateurs et de recueillir leurs avis et remarques afin d'améliorer le site
      • Contraintes : Accès unique à partir de la page de connexion Look&Note

    • Confidentialité :

      • Description : Définir les règles de confidentialité que le site Look&Note s'engage à respecter
      • Priorité : Obligatoire
      • Justification : L'objectif est de faire connaître auprès des utilisateurs la politique du site en indiquant le type de données qu'il recueille et l'utilisation qu'il en fait
      • Contraintes : Accès unique à partir de la page de connexion Look&Note

    • Conditions d'utilisation :

      • Description : Définir les conditions générales d'utilisation (CGU) du site Look&Note
      • Priorité : Obligatoire
      • Justification : Interdire d'utiliser un compte Look&Note pour la conduite d'activités qui incluent, sans restreindre la portée générale de certaines pratiques
      • Contraintes : Accès unique à partir de la page de connexion Look&Note

3- Besoins non fonctionnels

    Les contraintes d''usage des fonctions.
    Les contraintes de conception et de performance.
    Qualités attendues du logiciel

  • Couleurs et styles :

    • Description :Utilisation de couleurs agréables et attirantes, le bandeau supérieur devra être dans un orange dégradé, la partie inférieure sera en fond blanc avec un mélange de couleurs modernes
    • Priorité : Obligatoire
    • Justification : Rendu visuel agréable, attirant et moderne
    • Développement : Affichage avec des styles CSS et des balises HTML

  • Base de données :

    • Description :Développer une base de données avec des tables pour enregistrer les données des comptes utilisateurs, à savoir : le nom d'utilisateur, le mot de passe, l'adresse mail, les photos publiées, les critères de jugement, les commentaires, les notes données, les notes attribuées, la moyenne des notes données, la moyenne des notes obtenues, le nombre de notes données, le nombre de notes obtenues et le nombre de publications
    • Priorité : Obligatoire
    • Justification : Enregistrement des données personnelles de l'utilisateur
    • Développement : La base de données est développée en MySQL

  • Pages web:

    • Description :Développer des pages web contenant des liens sur d'autres pages web et dans lesquels il est possible de faire des recherches par mots clés
    • Priorité : Obligatoire
    • Développement : Les pages web sont développées avec du code HTML et des styles CSS, elles sont gérées avec du code PHP qui permet également d'encapsuler des requêtes MySQL pour la gestion des bases de données
